💡 principles

a few things i've learned the hard way about automation, gtm, and shipping.

  • automation isn't a feature, it's a stance. if you're doing something twice a week and it has rules, write the rules down.
  • the bottleneck is almost never the tool. it's the workflow nobody wrote down.
  • a system you can edit beats a tool you can't.
  • ship the ugly version on monday. iterate the rest of the week.
  • gtm engineering is business automation for go-to-market. the rebrand was earned, but the craft is old.
  • data hygiene compounds. so does data debt.
  • the best outbound looks like a normal conversation that happens to have great timing.
